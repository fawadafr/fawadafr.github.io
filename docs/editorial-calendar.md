## Status Legend

- **draft** - Work in progress in `_drafts/` folder
- **scheduled** - In `_posts/` with future date
- **published** - Live on site
- **idea** - Planned but not started

## Publishing Workflow

### 1. Ideation
- Add post title to calendar with "idea" status
- Define target publish date
- Assign topic/tags

### 2. Drafting
- Create file in `_drafts/` folder
- Filename: `post-title-slug.md` (no date prefix)
- Preview locally: `bundle exec jekyll serve --drafts`
- Update status to "draft"

### 3. Scheduling
- Move draft to `_posts/` folder
- Add date prefix: `YYYY-MM-DD-post-title-slug.md`
- Set future date in filename
- Update status to "scheduled"
- GitHub Pages will publish automatically after the date

### 4. Publishing
- For immediate publish: use today's date or past date
- For scheduled: future date (will publish on next rebuild after that date)
- Commit and push to deploy
- Update status to "published"

## Content Guidelines

### Post Structure
- **Title:** Clear, action-oriented (60 chars max for SEO)
- **Excerpt:** 1-2 sentences summarizing the post (150 chars)
- **Tags:** 2-5 relevant tags from established taxonomy
- **Length:** 800-2000 words for thought leadership pieces
- **Images:** Include header image if available

### Tags Taxonomy
- **Leadership:** leadership, management, culture, hiring, scaling
- **Technical:** platform-engineering, cloud-architecture, distributed-systems, devops
- **Strategy:** technical-strategy, innovation, decision-making
- **Culture:** engineering-culture, developer-experience, productivity

### Publishing Cadence
- **Target:** 2-3 posts per month (bi-weekly to weekly)
- **Best Days:** Tuesday-Thursday for maximum engagement
- **Time:** Morning (9-11am PT) for optimal readership
